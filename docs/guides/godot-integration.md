# Godot integration — how the router talks to the engine

Two transports, one router. This guide explains both paths, when each is used, and what to check
when something looks broken.

## Architecture at a glance

```
┌──────────────────────┐    stdio (JSON-RPC)    ┌────────────────────────────┐
│  Cursor / MCP client │ ─────────────────────▶ │ Terravolt MCP router       │
│  (terravolt-godot-mcp)│ ◀───────────────────── │ packages/mcp-server/dist   │
└──────────────────────┘                        └─────────────┬──────────────┘
                                                              │
        ┌──────────────────────── editor mode ────────────────┤
        │                                                      │
        │ ws://127.0.0.1:6505 (JSON-RPC 2.0)                   │
        ▼                                                      │
┌────────────────────────────────────┐                        │
│ Godot editor + Terravolt addon     │ persistent             │
│ packages/godot-mcp-addon/main.gd   │ socket                 │
│ (mcp_server.gd → dispatcher.gd)    │                        │
└────────────────────────────────────┘                        │
                                                              │ headless mode
                                                              │ (WS unreachable +
                                                              │  headlessFallback)
                                                              ▼
                                          spawn `godot --headless --path <proj>
                                            --script headless_driver.gd`
                                          parse `TERRAVOLT_HEADLESS_PORT=<n>`
                                          ┌───────────────────────────────┐
                                          │ TCP loopback JSON-RPC driver  │
                                          │ (process lifetime managed by  │
                                          │  HeadlessCoordinator)          │
                                          └───────────────────────────────┘
```

## Editor mode

Used when the Godot editor is open and the Terravolt addon is enabled.

1. **Daemon listens** — `mcp_server.gd` binds a WebSocket server on `127.0.0.1:6505` (configurable
   in **Project Settings → terravolt_mcp**).
2. **Router connects** — `GodotWsClient` opens the socket on startup, reconnects with exponential
   backoff if it drops, sends WS pings every `heartbeat-interval-ms` (15 s default), and kills the
   socket if no pong arrives within `heartbeat-timeout-ms` (45 s default).
3. **Dispatch** — every daemon-bridged MCP tool maps to a JSON-RPC method (see
   `packages/shared/methods/registry.json`). Input is AJV-validated by the router before send.
4. **Notifications flow back** — daemon-emitted notifications are re-broadcast to the MCP client as
   `notifications/message` (`logger: "terravolt.daemon.<method>"`).
5. **Cancellation** — when the MCP client aborts a tool call, the router sends
   `dispatch.cancel { target_id: <id> }` as a JSON-RPC notification to the daemon.

### What lives in the addon

| File                          | Role                                                                                |
| ----------------------------- | ----------------------------------------------------------------------------------- |
| `main.gd`                     | `EditorPlugin`. Wires the bottom panel + lifecycle.                                 |
| `mcp_server.gd`               | WebSocket server, peer management, heartbeat.                                       |
| `dispatcher.gd`               | JSON-RPC parser + method registry + error envelope.                                 |
| `logging.gd`                  | Rotating file logger writing `user://mcp_log.txt`.                                  |
| `error_codes.gd`              | Stable `-33xxx` application codes mirroring `packages/shared/errors/registry.json`. |
| `_generated/catalog_meta.gd`  | Auto-generated by `npm run catalog:sync`.                                           |
| `headless/headless_driver.gd` | TCP driver used by headless mode (loaded via `--script`, not as part of the addon). |

### Enabling the addon

```powershell
# Stage the addon into your Godot project once:
$env:TERRAVOLT_GODOT_PROJECT = "C:\path\to\my-godot-project"
npm run addon:link
```

In Godot: **Project → Project Settings → Plugins → Terravolt MCP → Enable.** The bottom panel
**Terravolt MCP** then shows status / start / stop / log.

## Headless mode

Used by the `headless.*` MCP tools, and as **automatic fallback** for any daemon-bridged tool whose
registry row sets `headlessFallback: true` (currently `ping` and `server.info`).

1. **Resolve binary.** `resolveGodotBinary` checks `--godot-binary`, `TERRAVOLT_GODOT_BINARY`,
   `PATH`, and platform install dirs (`%LOCALAPPDATA%\Programs\Godot\**` on Windows,
   `/Applications/Godot.app` on macOS, `~/.local/share/godot/**` on Linux). On Windows it prefers
   `*_console.exe` because stderr (used for the port handshake) is reliable.
2. **Resolve project.** `headless.start_project.arguments.projectPath` takes precedence, then
   `--project`, then `TERRAVOLT_PROJECT_PATH`.
3. **Spawn.** `launchHeadlessDriver` calls:
   ```text
   godot --headless --path <project> --script <abs path to headless_driver.gd>
   ```
   with `TERRAVOLT_CATALOG_VERSION` and `TERRAVOLT_REGISTRY_SHA256` injected as environment
   variables so `server.info` reports the right catalog.
4. **Handshake.** The driver opens a TCP listener on an ephemeral port and emits
   `TERRAVOLT_HEADLESS_PORT=<n>` to stderr. The router parses it.
5. **RPC.** Newline-delimited JSON-RPC 2.0 over loopback TCP. The driver exposes a small allowlist:
   `ping`, `server.info`, `server.list_methods`, `script.validate_syntax`, `dispatch.cancel`.
6. **Shutdown.** `headless.stop` sends `SIGTERM` (or `SIGKILL` with `force: true`); the router also
   stops the session when it shuts down.

### Why not just keep the editor open?

You can — the headless path exists for:

- **CI and Cursor cloud agents** where no display is available.
- **GDScript compile checks** during large refactors without paying for a full editor boot.
- **Resilience** — the router never returns "editor offline" for `ping` / `server.info` because it
  can fall back.

### Limitations

- Headless does **not** load `EditorPlugin`s and cannot exercise editor-only APIs (`scene.*`,
  `node.*`, `inspector.*`, etc.). Expansion is tracked in `docs/tasklist/07-headless-fallback.md`
  and `docs/catalog/parity.md`.
- C# (`.cs`) compile checks are GDScript-only today; .NET parity is scheduled for §08.

## Verifying the connection

| Test               | Command                                                 | Expected                           |
| ------------------ | ------------------------------------------------------- | ---------------------------------- |
| Godot binary works | `& $env:TERRAVOLT_GODOT_BINARY --version`               | `4.x.x.stable[.mono]…`             |
| Router boots       | `node packages/mcp-server/dist/index.js --print-config` | JSON config on stderr              |
| Catalog matches    | call `tools.health` from any MCP client                 | `pass: true`                       |
| Editor reachable   | call `ping`                                             | `method: "ping"` (not `@headless`) |
| Headless reachable | call `headless.start_project` → `headless.status`       | `alive: true`                      |
| GDScript pipeline  | call `headless.validate_script` on a known-good `.gd`   | `result.ok: true`                  |

Most of these are exercised end-to-end by `packages/mcp-server/tests/integration/mcp_e2e.test.mjs`
and `addon_parse.test.mjs` against a real Godot 4.6.3 stable mono binary.

## Common pitfalls

- **`transport.not_connected` on every call**: the editor isn't running with the addon enabled.
  Check that **Terravolt MCP** is enabled in Project Settings → Plugins, or rely on
  `headless.start_project` for tools that support the fallback.
- **`headless.binary_missing` (`-33810`)**: run `npm run env:godot`. On Windows, prefer
  `_console.exe`.
- **`headless.driver_handshake_failed` (`-33813`)**: stderr from Godot isn't reaching the router —
  usually the non-console exe on Windows.
- **Catalog mismatch**: `tools.health` flags `protocol_catalog_mismatch_detected: true`. Run
  `npm run catalog:sync` and restart both processes.
