# TerraVolt MCP — Godot 4 addon

Editor plugin exposing a **localhost WebSocket JSON-RPC daemon** (`:6505` by default) plus
structured logging to **`user://mcp_log.txt`**.

The same folder also ships the **headless TCP driver** (`headless/headless_driver.gd`) — loaded by
the Node router via `godot --headless --script <abs path>` and **independent** of the addon being
mounted in the target project.

## Requirements

- **Godot 4.x** (tested against 4.6.3 stable mono). `.NET`-compatible builds are fine; the addon
  itself is GDScript-only.
- This folder is symlinked / junctioned / copied into a dev Godot project as `addons/terravolt_mcp/`
  (see linking below).

## Link into a Godot dev project

From repo root:

1. Set `TERRAVOLT_GODOT_PROJECT` to the absolute path of your Godot project **or** put
   `{ "godotProject": "<absolute/path>" }` in `~/.terravolt-mcp-dev.json` (Windows:
   `%USERPROFILE%\.terravolt-mcp-dev.json`).
2. Run:

```bash
npm run addon:link
```

Overwrite an existing addon dir:

```bash
npm run addon:link -- --force
```

Remove:

```bash
npm run addon:unlink
```

## Editor usage

Enable **TerraVolt MCP** in **Project Settings → Plugins**. Use the bottom panel **TerraVolt MCP**
for status, ledger, Start/Stop/Restart, log actions. TerraVolt settings namespace:
`terravolt_mcp/...` (port, bind, heartbeat, logging, etc.).

## Phase map

Implemented per **`docs/tasklist/02`**–**`04`**:

- **`02`** — plugin shell, bottom panel, project settings defaults, link scripts.
- **`03`** — `TCPServer` + `WebSocketPeer` listener, single-client policy, heartbeat. Default
  `heartbeat_mode=control_frame` drives native WS ping via `WebSocketPeer.set_heartbeat_interval`;
  use `rpc` or `both` for JSON-RPC `server.heartbeat` notifications plus data-idle timeout pruning.
- **`04`** — JSON-RPC 2.0 dispatch, TerraVolt error envelope, rotation logger, RPC plumbing methods
  (`ping`, `echo`, `server.info`, `log.tail`, …).

## Headless driver notes

The driver is **self-contained** — no `preload("...")` of sibling addon scripts, because the router
can launch it against a project that doesn't mount the addon at all. Catalog metadata
(`catalog_version`, `registry_sha256`) arrives via env vars (`TERRAVOLT_CATALOG_VERSION`,
`TERRAVOLT_REGISTRY_SHA256`) injected by the Node router on spawn.

Supported methods: `ping`, `server.info`, `server.list_methods`, `script.validate_syntax`,
`dispatch.cancel`.

## Automated tests

- `packages/mcp-server/tests/integration/addon_parse.test.mjs` stages this folder as
  `tests/_fixtures/with-addon/addons/terravolt_mcp/` and runs `godot --headless --import` to verify
  every `.gd` parses with full `class_name` resolution.
- `packages/mcp-server/tests/integration/headless.test.mjs` exchanges `ping` + `server.info` against
  the driver over real TCP.

**GUT** is the planned harness for in-engine unit tests (installation / deferred harness wiring in
**`docs/tasklist/10`**).
